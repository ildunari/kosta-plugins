# iOS .gitignore

The canonical `.gitignore` for iOS projects. Copy everything between the fences into your `.gitignore` file.

Each section is annotated with why those files are excluded. Understanding the "why" helps you decide whether to add exceptions for your specific project.

```gitignore
# ==============================================================================
# Xcode
# ==============================================================================

# User-specific Xcode settings. These contain your personal window layout,
# breakpoints, and scheme preferences. They change constantly and cause
# merge conflicts if tracked.
*.xcuserstate
xcuserdata/

# Shared scheme data is intentionally NOT excluded. Shared schemes let your
# team use the same build/test/run configurations. Only user schemes are excluded.
# If using XcodeGen, you can exclude all schemes since they're regenerated:
# *.xcscheme

# Xcode project files -- ONLY exclude if using XcodeGen.
# XcodeGen regenerates these from project.yml, so they're build artifacts.
# Remove these lines if you're NOT using XcodeGen.
*.xcodeproj

# Xcode workspaces created by CocoaPods or SPM. Exclude if generated.
# Keep if you maintain the workspace manually.
# *.xcworkspace

# ==============================================================================
# Build Artifacts
# ==============================================================================

# Xcode's build cache. This is the #1 cause of "my repo is 5GB" problems.
# It contains compiled objects, intermediate files, and build logs.
# Xcode regenerates it on every build.
DerivedData/

# Swift Package Manager build directory.
.build/

# Build output directories (custom configurations).
build/
Build/

# Index data used by Xcode for code completion. Regenerated automatically.
IndexStores/

# ==============================================================================
# Swift Package Manager
# ==============================================================================

# SPM dependency checkouts. These are downloaded on `xcodebuild -resolvePackages`
# or when Xcode opens the project. Tracking them bloats the repo.
.swiftpm/
Packages/
Package.resolved

# NOTE: Some teams DO track Package.resolved to lock dependency versions.
# If you want reproducible builds (recommended for production apps),
# remove the Package.resolved line above and track it.

# ==============================================================================
# CocoaPods (if applicable)
# ==============================================================================

# The Pods directory contains downloaded CocoaPod dependencies.
# Like node_modules in JavaScript -- downloaded, not authored.
# Some teams track Pods/ for reproducible builds without `pod install`.
Pods/

# CocoaPods generates this workspace to wire pods into your project.
# *.xcworkspace  # Uncomment if using CocoaPods

# ==============================================================================
# Carthage (if applicable)
# ==============================================================================

# Carthage builds pre-compiled frameworks here.
Carthage/Build/
Carthage/Checkouts/

# ==============================================================================
# Code Signing
# ==============================================================================

# Provisioning profiles and certificates should NEVER be in version control.
# They contain sensitive identity information.
*.mobileprovision
*.provisionprofile

# Signing-related plists generated by Xcode.
ExportOptions.plist

# ==============================================================================
# Fastlane (CI/CD)
# ==============================================================================

# Fastlane output directory -- build artifacts, screenshots, logs.
fastlane/report.xml
fastlane/Preview.html
fastlane/screenshots/**/*.png
fastlane/test_output/

# ==============================================================================
# CI/CD Artifacts
# ==============================================================================

# Test result bundles. These can be huge (100MB+) and are generated fresh each run.
*.xcresult

# Code coverage reports.
coverage/
*.profdata

# Archive files (the .xcarchive produced by Xcode's Archive action).
*.xcarchive

# IPA files (the installable app package).
*.ipa

# dSYM files (debug symbol files for crash reporting).
# These are generated during archive and uploaded to crash reporting services.
*.dSYM
*.dSYM.zip

# ==============================================================================
# OS and Editor Files
# ==============================================================================

# macOS Finder metadata. Created when you browse folders in Finder.
.DS_Store

# Thumbnail cache files.
._*

# Spotlight indexing metadata.
.Spotlight-V100
.Trashes

# Vim swap files.
*.swp
*.swo
*~

# VSCode settings (unless your team standardizes on VSCode).
.vscode/

# JetBrains IDE settings (AppCode, etc.).
.idea/

# ==============================================================================
# Secrets and Environment Files
# ==============================================================================

# NEVER commit secrets. API keys, tokens, passwords.
.env
.env.*
*.secret
*.key

# Secrets management tools.
secrets/

# ==============================================================================
# Miscellaneous
# ==============================================================================

# Playground data -- Xcode Playground build artifacts.
timeline.xctimeline
playground.xcworkspace

# Tuist-generated files (if using Tuist instead of XcodeGen).
Derived/

# Compressed archives of the project.
*.zip
*.tar.gz

# XcodeGen cache.
.xcodegen/
```

## Quick Start

To create the `.gitignore` file in a new project:

```bash
# From the project root directory:
cat > .gitignore << 'GITIGNORE'
# Paste the content from above
GITIGNORE
```

Or if using the ios-project-wizard, the `.gitignore` is generated automatically during Phase 8.

## Common Modifications

**Track Package.resolved for reproducible builds:**
Remove the `Package.resolved` line. This locks your SPM dependency versions so every developer and CI machine builds with the exact same versions.

**Track Pods/ for faster CI:**
Remove the `Pods/` line. CI won't need to run `pod install`, saving 1-3 minutes per build. Tradeoff: your repo gets bigger.

**Not using XcodeGen:**
Remove the `*.xcodeproj` line. You need to track the project file if you're not generating it.

**Using Xcode Cloud:**
Add `ci_scripts/` to track your CI scripts. Xcode Cloud looks for `ci_scripts/ci_post_clone.sh` for custom setup.
