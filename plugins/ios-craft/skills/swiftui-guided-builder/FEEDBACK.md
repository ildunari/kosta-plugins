# swiftui-guided-builder Feedback Log

Track what works and what doesn't when this skill runs. Update after each use.

## Format

```
## YYYY-MM-DD — [outcome: success | partial | failure]
- **What was built:** [screen type + one-line description]
- **What worked well:** [specific pattern, template, or decision that landed]
- **What missed:** [wrong state management choice, missing a11y, template mismatch]
- **False positive/negative:** [checklist item that flagged incorrectly or missed a real issue]
- **Adjustment made:** [if you changed the skill based on this, note it here]
```

## Tracking Goals

Over time, this log reveals:
- Which screen type templates get used as-is vs heavily modified (refine the weak ones)
- Whether the state management decision tree picks correctly (track overrides)
- Whether the a11y checklist catches real issues or generates noise
- Which "Common Mistakes to Catch" actually fire in practice
- Whether iOS 17+ defaults are correct or users still need iOS 16 patterns
